# sources.avar.me

Хранилище исходных данных для проектов [avar.me](https://avar.me): словари, книги, текстовые корпусы. Публикуется через GitHub Pages: <https://sources.avar.me>.

Это **источник правды** (single source of truth) для всех остальных сайтов и инструментов экосистемы avar.me. Файлы в `data/` нужно править здесь — пересборка автоматически обновит и сам сайт, и зависимые проекты, которые подтянут данные при следующей сборке.

## Что внутри

- **`sources.json`** — реестр всех исходников: id, заголовок, описание, формат, путь, версия, лицензия.
- **`data/`** — собственно данные. Сейчас два словаря в `jsonl`:
  - `av-ru.jsonl` — аварско-русский (~22 800 статей)
  - `ru-av.jsonl` — русско-аварский (~38 800 статей)
- **`src/build_site.py`** — генератор статического сайта из `sources.json` + `data/`.
- **`docs/`** — собранный сайт (артефакт сборки; не редактировать вручную).

## Использование

### Просто почитать словари

Откройте <https://sources.avar.me> и переходите в нужный словарь. Навигация по алфавиту, как в книге.

### Сообщить об ошибке

Заметили опечатку, неверный перевод, кривое расширение формы? Напишите в Telegram-канал [@avarlangme](https://t.me/avarlangme) — там же обсуждаем все правки.

### Скачать сырые данные

Прямые ссылки (стабильные, на них опираются другие проекты):

- <https://sources.avar.me/data/av-ru.jsonl>
- <https://sources.avar.me/data/ru-av.jsonl>
- <https://sources.avar.me/sources.json>

### Подтянуть данные в свой проект

```bash
curl -fsSL https://sources.avar.me/data/av-ru.jsonl > av-ru.jsonl
```

Так делают [forms](https://github.com/avar-me/forms), [avar.me](https://avar.me), `corrector` и прочие — данные **не дублируются** между проектами.

## Формат данных (jsonl)

Одна строка = одна статья словаря. Пример:

```json
{
  "word": "бакъ",
  "stress": 1,
  "stem": "бакъ",
  "forms": ["бакълъи", "бакълъул"],
  "senses": [
    {
      "text": "солнце",
      "labels": ["существительное"],
      "examples": [
        {"av": "бакъ борхьинчӏого", "ru": "до захода солнца"}
      ]
    }
  ],
  "see_also": [{"target": "бакъул", "kind": "see"}]
}
```

Поля (необязательные могут отсутствовать):

| Поле | Описание |
|------|----------|
| `word` | Заглавное слово. |
| `stress` | Позиция ударной гласной (1-based) или отсутствует. |
| `stem` | Основа для раскрытия форм. |
| `forms` | Раскрытые словоформы. |
| `senses[]` | Значения. У каждого: `text`, `comment`, `labels[]`, `examples[]`. |
| `senses[].examples[]` | `{av, ru, labels?}` — примеры с переводом. |
| `see_also[]` | Ссылки «см.» / «от …»: `{target, kind}`. |

## Сборка сайта

```bash
./build.sh
```

Локально проверить:

```bash
python3 -m http.server -d docs 8000
# открыть http://localhost:8000
```

Деплой автоматический: `git push origin main` → GitHub Actions собирает `docs/` и публикует на GitHub Pages.
