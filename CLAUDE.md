# Инструкции для AI-ассистентов в этом репозитории

Этот файл — общая инструкция для Claude, Copilot, Cursor и других. Если вы агент — прочитайте до начала любых изменений.

## Что это за репозиторий

`sources.avar.me` — **источник правды** для всех данных экосистемы avar.me (словари, книги, корпусы). Сайт собирается из этих же данных и публикуется на GitHub Pages.

Другие репозитории (`forms`, `avar.me`, `corrector`, `index` и т. д.) при сборке выкачивают `data/*.jsonl` отсюда. **Они не правят данные у себя — они потребители.** Любые правки данных должны быть здесь.

## Главное правило

1. **Данные** (`data/*.jsonl`) — священная корова. Правки строго по запросу пользователя, обычно после обсуждения в Telegram-чате [@avarme_chat](https://t.me/avarme_chat).
2. **Не плодите данные.** Если в `data/` уже есть `av-ru.jsonl`, не создавайте `av-ru-v2.jsonl` — правьте существующий.
3. **Оригиналы (PDF) — в `books/`.** Большие исторические сканы и оригиналы; ссылки на них в `sources.json` → `documents[]`. Не дублировать, не пересжимать.
4. **Не редактируйте `docs/`.** Это артефакт сборки. Правьте `src/build_site.py` или шаблоны.
5. **Не делайте автоматическую пересборку при правке шаблонов вручную в `docs/`** — деплой через GitHub Actions сделает это сам после `git push`.

## Аварская графика

Аварский — на кириллице с диграфами: **тӏ, лъ, гь, гъ, гӏ, кь, къ, кӏ, хь, хъ, хӏ, цӏ, чӏ**. В `src/build_site.py` есть аварский токенизатор: он считает каждый диграф одной буквой. Это важно:

- Алфавит в навигации содержит диграфы как отдельные «буквы».
- Подстраницы группируются по **трём аварским буквам** слова, поэтому `бет` и `бетӏ` оказываются на разных страницах (4 символа, но 3 буквы — третья «тӏ»).

### Правила нормализации палочки

`normalize_palochka()` (в `src/build_site.py` и в парсерах источников) приводит палочку к каноническому виду по этим правилам:

1. **Палочка может стоять только внутри диграфа.** Допустимые основы: **т, г, к, х, ц, ч** (любого регистра). Всё, что выглядит как палочка, но не следует за этими буквами, — это не палочка.
2. **Регистр палочки совпадает с регистром основы:** строчная `ӏ` (U+04CF) после строчной буквы, заглавная `Ӏ` (U+04C0) после заглавной (`ГӀумру`, `гӏумру`, но не `Гӏумру` или `гӀумру`).
3. **Палочка может приходить разными глифами** в источнике: `Ӏ`, `ӏ`, `|`, `ǀ`, латинские `I`, `i`, `l` и даже цифра `1` (типист быстро печатает). Конвертируется в палочку только если следует за диграф-основой; иначе:
   - Палочка-глиф (`Ӏ`, `ӏ`, `|`, `ǀ`) вне диграфа — это ошибка типиста; меняется на цифру `1` (годы вроде `Ӏ993` → `1993`, нумерация `Ӏ классалдаса` → `1 классалдаса`).
   - Цифра `1` вне диграфа остаётся цифрой (`1993`, `11.08`, `1 класс`).
   - Латинские `I`, `i`, `l` вне диграфа: если в окружении нет других латинских букв (т. е. сидят среди кириллицы/пунктуации/цифр), считаются ошибочной цифрой `1`; иначе — это легитимная латиница в иноязычном слове и не трогается.
4. **Удвоение палочки в диграфе — типист дважды нажал клавишу.** После эмиссии палочки «съедаем» подряд идущие палочка-кандидаты (`чII` → `чӏ`, `т11` → `тӏ`, `кӏI` → `кӏ`).

Для русско-аварского словаря (`ru-av`) исходный язык — русский, без диграфов. Используется стандартный русский алфавит.

## Поток работы

```
data/*.jsonl  ──►  src/build_site.py  ──►  docs/  ──►  GitHub Pages  ──►  sources.avar.me
                          ▲
                  src/templates/*
                  sources.json
                  books/*.pdf
```

1. Пользователь сообщает об ошибке в данных в чате [@avarme_chat](https://t.me/avarme_chat).
2. Админ правит соответствующую строку в `data/<source>.jsonl`.
3. **Добавляет запись в [`CHANGELOG.md`](CHANGELOG.md)** (см. ниже).
4. `git commit && git push` — Actions пересобирает сайт.
5. Другие сайты подтянут обновлённый jsonl при своей следующей сборке.

## Changelog (обязательно)

Любая смысловая правка данных (`data/*.jsonl`, `data/*.json`) **должна** сопровождаться записью
в [`CHANGELOG.md`](CHANGELOG.md): дата, краткое описание, затронутый источник (`av-ru`, `ru-av`, …)
и объём (число статей/строк). Новые записи — сверху. Это единственный человекочитаемый журнал
для потребителей данных (`forms`, `avar.me`, `corrector`, `index`); jsonl-дифф листать тяжело.

Для крупных/массовых правок дополнительно кладём подробный построчный diff в папку
[`changelog/`](changelog/) (`ГГГГ-ММ-ДД-<источник>-<тема>.md`) и ссылаемся на него из записи в
`CHANGELOG.md` — чтобы корректоры могли сверить каждую статью с источником. Формат строк:
`text[i]: «было» -> «стало»`, `- удалённый пример`, `+ добавленный пример (av = ru [метки] (коммент))`.

Служебные правки (сайт, шаблоны, CI, `docs/`) в changelog не заносим.

## Виды источников (`kind`)

В `sources.json` у каждого источника есть поле `kind`:

- **`dictionary`** — словарная статья на статью; формат `jsonl` (одна запись = одно слово). Сайт строит алфавит → 3-буквенные подстраницы со статьями в виде raw JSON.
- **`corpus`** — параллельный корпус (например, повесть `taras_bulba.json`). Формат `json` (массив элементов с `chapter`, `SentenceNumber`, `av`, `ru`). Сайт строит главу → страницу с параллельным `ru ↔ av` списком предложений; нумерация идёт по главам и `SentenceNumber`.

При добавлении нового источника указывайте правильный `kind` — это определяет, какая ветка сборки в `src/build_site.py` отработает.

## Что менять, а что нет

| Хочется поменять | Куда лезть |
|------------------|-----------|
| Опечатку в слове | `data/<source>.jsonl` (найти `word: ...`) |
| Перевод / пример | `data/<source>.jsonl` |
| Опечатку в корпусе | `data/<source>.json`, найти запись по `chapter` + `SentenceNumber` |
| Новый источник | Добавить запись в `sources.json` + положить файл в `data/` |
| Внешний вид сайта | `src/templates/styles.css`, `src/templates/app.js` |
| Логика рендера страниц | `src/build_site.py` |
| Список словарей на главной | автоматически из `sources.json` |
| Доменное имя | `CNAME` |
| Деплой | `.github/workflows/deploy.yml` |

## Стиль

- Сайт — статический. Никакого SSR, никакого билд-инструмента сложнее одного python-скрипта. Никаких внешних JS-зависимостей через npm. Шрифты через Google Fonts (как в `index.avar.me`).
- Никаких эмодзи в шаблонах и текстах. Кроме SVG `☀` в favicon — он часть бренда avar.me.
- Сохраняйте дизайн в линии с [index.avar.me](https://index.avar.me) и [dev.avar.me](https://dev.avar.me): мягкий кремовый фон, цвет акцента `--accent: #9a7b1a`, шрифты Onest (UI) + Literata (заголовки и сериф).

## Когда сомневаетесь

Спросите перед тем как:

- Делать массовые замены в `data/*.jsonl` (regex по всему файлу).
- Менять формат данных (новые/убирать поля).
- Удалять записи.
- Менять имена путей или URL — это сломает потребителей (forms, avar.me, corrector).
